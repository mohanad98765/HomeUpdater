## الدليل الشامل للمشروع

---

## 1. الرؤية (The Vision)

بناء برنامج واحد، يعمل على جهاز كمبيوتر واحد في المنزل (Hub)، يقوم تلقائيًا بـ:

1. **اكتشاف** كل الأجهزة المتصلة بالشبكة المنزلية
2. **تحديد** نوع كل جهاز ومصنّعه وإصداره الحالي
3. **البحث** عن أحدث إصدارات السوفت وير والفريم وير له من **كل المصادر**:
   - المصادر الرسمية (موقع الشركة، Windows Update، Apple، إلخ)
   - المصادر غير الرسمية (GitHub، XDA، OpenWRT، مجتمعات Custom Firmware)
4. **عرض** قائمة شاملة بكل التحديثات المتاحة، مع توضيح:
   - مصدر التحديث (رسمي / مجتمع)
   - تاريخ الإصدار
   - ملاحظات الإصدار (Changelog)
   - مستوى الموثوقية (موصى به / تجريبي / خطر)
5. **تنفيذ** التحديث تلقائيًا في الأجهزة اللي تسمح بذلك، **بإذن المستخدم**

---

## 2. لماذا هذا المشروع موجود (Why)

اليوم، أي شخص في منزله عنده 20-50 جهازًا متصلًا بالإنترنت. كل واحد منها:
- يحتاج تحديث ولكن من مكان مختلف
- بعضها لا يحدّث نفسه أبدًا (راوترات، IoT)
- بعضها يحتوي ثغرات أمنية معروفة لا يعرفها المستخدم
- لا يوجد لوحة تحكم موحّدة تجمعها كلها

**النتيجة:** أجهزة قديمة بثغرات، شبكة منزلية ضعيفة، وتجربة مستخدم سيئة.

**الحل:** Hub موحّد يفعل كل شيء من مكان واحد.

## 3. النطاق (Scope)

### ما يفعله البرنامج (In Scope)

| الفئة | الاكتشاف | عرض التحديث | تنفيذ التحديث |
|-------|----------|--------------|----------------|
| كمبيوتر Windows | ✅ | ✅ | ✅ تلقائي (winget) |
| كمبيوتر Linux | ✅ | ✅ | ✅ تلقائي (SSH + apt) |
| كمبيوتر macOS | ✅ | ✅ | ⚠️ شبه تلقائي |
| جوال أندرويد | ✅ | ✅ | ⚠️ ADB أو تنبيه |
| جوال iOS | ✅ | ⚠️ تنبيه فقط | ❌ مستحيل |
| راوتر مفتوح (OpenWRT/pfSense) | ✅ | ✅ | ✅ تلقائي |
| راوتر مغلق (مزوّد الخدمة) | ✅ | ⚠️ تنبيه فقط | ❌ مستحيل |
| تلفزيون Android TV | ✅ | ✅ | ⚠️ ADB |
| تلفزيون Tizen/webOS | ✅ | ⚠️ تنبيه فقط | ❌ مستحيل |
| كاميرات IP | ✅ | ✅ (في بعضها) | ⚠️ جزئي |
| أجهزة Smart Home (Hue, etc.) | ✅ | ✅ | ✅ عبر Home Assistant |
| طابعات | ✅ | ✅ (في بعضها) | ⚠️ جزئي |

### ما لا يفعله البرنامج (Out of Scope)

- ❌ لن يكسر قيود iOS أو Tizen — هذه قيود مفروضة من المصنّع
- ❌ لن يحدّث أجهزة معطّلة أو خارج الشبكة
- ❌ لن يضمن نجاح كل تحديث (firmware updates يمكن أن تفشل)
- ❌ ليس بديلًا عن النسخة الاحتياطية (Backup)

## 4. الجمهور المستهدف

- **المستخدم الأساسي:** صاحب منزل يهتم بأمن شبكته (مثلك)
- **المستوى التقني المطلوب:** متوسط — يقدر يفتح متصفح ويفتح إعدادات الراوتر
- **النشر النهائي:** مفتوح المصدر للمجتمع (احتمال)

---

## 5. الحدود التقنية الصارمة (Hard Limits)

هذه ليست تحديات نتغلب عليها، هذه قيود مفروضة من خارجنا:

1. **Apple ecosystem (iOS/iPadOS/tvOS):**
   - لا يوجد API محلي للتحديث
   - أي تحديث يجب أن يمر عبر Apple نفسها
   - أقصى شيء نقدر نسويه: نقرأ إصدار الجهاز ونقول للمستخدم "حدّث يدويًا"

2. **تلفزيونات Samsung Tizen و LG webOS:**
   - مغلقة بالكامل
   - لا API علني للتحديث
   - تنبيه فقط

3. **راوترات بعض المزوّدين (STC, Mobily, Du):**
   - الـ firmware مقفّل من المزوّد
   - لا يوجد طريقة قانونية لتجاوزه
   - تنبيه فقط

4. **أجهزة IoT الرخيصة المجهولة المصدر:**
   - أحيانًا لا يوجد لها أي تحديثات
   - أحيانًا التحديث الوحيد عبر تطبيق صيني
   - معالجة حالة بحالة

## 6. خارطة الطريق (Roadmap)

### Phase 1 - الأساس (الأسابيع 1-2)
- اكتشاف الشبكة (Network Scan)
- قاعدة بيانات الأجهزة (Inventory)
- لوحة تحكم Web UI بسيطة
- دعم Windows + Linux في التحديث

### Phase 2 - التوسع (الأسابيع 3-5)
- دعم راوترات OpenWRT/pfSense
- ربط Home Assistant للأجهزة الذكية
- جلب CVE feeds وعرض الثغرات
- إشعارات (تطبيق جوال أو إيميل)

### Phase 3 - الإثراء (الأسابيع 6-8)
- جلب التحديثات غير الرسمية (GitHub, XDA)
- ADB integration للأندرويد
- جدولة تلقائية للتحديثات
- تقارير دورية

### Phase 4 - النضج (الأسابيع 9+)
- توسيع قاعدة البيانات لمصنّعين أكثر
- نظام Plugins للأجهزة الجديدة
- نشر مفتوح المصدر

### Phase A (متوازي) - الهوية البصرية
- شعار، ألوان، خطوط
- تصميم mockups لكل صفحة
- نظام تصميم متّسق (راجع BRAND.md)

### Phase B (متوازي/متأخر) - الـ Installer
- بناء Installer شامل بـ Inno Setup
- تضمين كل المتطلبات (Python, Nmap, إلخ)
- شاشات تثبيت احترافية بالألوان والشعار
- يعمل بنقرة-مزدوجة بدون أي متطلبات سابقة

## 7. معايير النجاح (Definition of Done)

نعتبر المشروع ناجحًا في v1 عندما:
- ✅ يكتشف 90%+ من أجهزة الشبكة المنزلية تلقائيًا
- ✅ يعرض تحديثات السوفت وير والفريم وير لكمبيوترات Windows/Linux تلقائيًا
- ✅ يعرض تنبيهات لكل جهاز لا نقدر نحدّثه تلقائيًا
- ✅ المستخدم يقدر يشغّله من أيقونة على Windows بدون سطر أوامر
- ✅ آمن: لا يخرّب أي جهاز بسبب تحديث خاطئ
- ✅ **يُسلَّم كملف Installer واحد** (.exe) يثبّت كل شيء تلقائيًا
- ✅ **هوية بصرية احترافية** على مستوى منتجات تجارية (راجع BRAND.md)
- ✅ يعمل على Windows 10 و 11 بدون متطلبات سابقة

## 8. الفلسفة العامة

1. **استخدم الموجود**: لا نعيد اختراع العجلة. كل مكوّن نختار له أداة جاهزة مفتوحة المصدر.
2. **آمن أولًا**: نحن نتعامل مع الفريم وير. التحديث الخاطئ يخرّب الجهاز. نتأكد ثم نتأكد.
3. **شفّاف**: المستخدم يجب أن يرى مصدر كل تحديث وقراره النهائي بيده.
4. **يعمل بدون اتصال بالإنترنت** للأجهزة المحلية على الأقل (بعد التحميل الأول).

## 9. كيف نتعامل مع المشروع

كل جلسة عمل تتبع هذا التسلسل:
1. **افتح PROGRESS.md** — اقرأ أين توقفنا
2. **افتح TASKS.md** — اختر مهمة من قائمة "Phase الحالية"
3. **اعمل عليها** — استخدم الأدوات المذكورة في ARCHITECTURE.md
4. **حدّث الملفات** — PROGRESS + DECISIONS إذا اتخذت قرارًا تقنيًا
5. **اعمل commit** — حتى لو ما عندك Git محلي، احفظ التغييرات في الملفات
