# التشغيل التلقائي ووضع الخدمة (Auto-start & Service)

كيف تجعل HomeUpdater يعمل تلقائياً، والفرق بين الخيارات وحدود كلٍّ منها.

---

## المشكلة
التطبيق يحتاج **صلاحيات مسؤول** (لتشغيل Windows Update / winget). هذا يخلق تعارضاً:
- اختصار مجلّد Startup لا يستطيع الرفع التلقائي للصلاحيات → يطلب UAC كل مرّة أو يفشل.
- خدمة تعمل كـ **LocalSystem** تكتب بياناتها في ملفّ SYSTEM (قاعدة/مفتاح منفصلان عن
  المستخدم)، **ولا تستطيع فكّ تشفير** اعتمادات المستخدم (مفتاح DPAPI مربوط بالمستخدم).

---

## ✅ المُوصى به: مهمّة مجدولة عند تسجيل الدخول
عند التثبيت، فعّل الخيار **«تشغيل HomeUpdater تلقائياً عند تسجيل الدخول»**. يُنشئ
المثبِّت **Scheduled Task** بخاصية «تشغيل بأعلى الامتيازات»:

```
schtasks /Create /TN "HomeUpdater" /TR "<app>\HomeUpdater.exe" /SC ONLOGON /RL HIGHEST /F
```

- يعمل **مرفوع الصلاحيات بلا نافذة UAC** عند كل تسجيل دخول.
- يعمل **بحساب المستخدم** → يشارك نفس البيانات ومفتاح التشفير (لا انقسام، والاعتمادات
  تُفكّ بشكل صحيح).
- إزالته تلقائية عند إلغاء التثبيت.

هذا يغطّي حاجة «يبدأ مع ويندوز» لمعظم المستخدمين.

---

## 🟡 خيار متقدّم: خدمة headless (بلا تسجيل دخول)
إن أردت تشغيل الـ backend **بدون أي تسجيل دخول** (خادم دائم)، استخدم `service.py`،
لكن **شغّلها بحساب المستخدم التفاعلي** (لا LocalSystem) ووجّهها لبيانات ذلك المستخدم:

```powershell
# كمسؤول، من مجلّد backend:
$env:HOMEUPDATER_DATA_DIR = "$env:APPDATA\HomeUpdater"   # شارك بيانات المستخدم
.\.venv\Scripts\python.exe service.py install --username .\<اسمك> --password <كلمتك>
.\.venv\Scripts\python.exe service.py start
# stop / remove للإيقاف / الإزالة
```

- `HOMEUPDATER_DATA_DIR` يجعل الخدمة والواجهة يتشاركان قاعدة/مفتاحاً واحداً (يحلّ
  انقسام البيانات).
- التشغيل بحساب المستخدم يُبقي مفتاح DPAPI صالحاً لفكّ الاعتمادات.
- **الحدود:** من بناء مُجمَّد (frozen) تحتاج exe خدمة منفصل (نقطة دخوله `service.py`)؛
  الـ exe المشحون هو الواجهة لا مضيف الخدمة. `service.py` يحتوي فرع مُرسِل SCM الصحيح
  للحالة المُجمَّدة، ويختار منفذاً حرّاً تلقائياً، ويُسجّل أخطاء الإقلاع في Event Log.

## مقارنة سريعة
| الخيار | يبدأ بلا تسجيل دخول | UAC | البيانات/المفتاح | التوصية |
|--------|:---:|:---:|------|---------|
| مهمّة مجدولة (logon) | ❌ (عند الدخول) | بلا نافذة | مُشتركة مع المستخدم ✅ | **الافتراضي** |
| خدمة كمستخدم + DATA_DIR | ✅ | — | مُشتركة ✅ | متقدّم |
| خدمة LocalSystem | ✅ | — | منفصلة ❌ (تعطّل فكّ الاعتمادات) | غير مُوصى به |
