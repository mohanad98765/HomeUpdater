# تشغيل واختبار تحديث Windows عن بُعد (WinRM)

دليل عملي لتفعيل WinRM على جهاز Windows آخر على شبكتك، ثم اختبار الفحص/الترقية
من HomeUpdater (تبويب «Windows بعيد»).

---

## 1) على الجهاز الهدف (الذي تريد تحديثه) — مرّة واحدة

افتح **PowerShell كمسؤول** على الجهاز الهدف ونفّذ:

```powershell
# 1. فعّل PowerShell Remoting (يُشغّل خدمة WinRM ويفتح المنفذ 5985)
Enable-PSRemoting -Force

# 2. اسمح بالاتصال من أجهزة الشبكة المحلية (إن لم يكن الجهاز في Domain)
#    استبدل النطاق بنطاق شبكتك:
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "192.168.1.*" -Force

# 3. تأكّد أن winget متوفّر (ثبّت "App Installer" من متجر Microsoft إن لزم)
winget --version
```

> **الشبكة عامّة (Public)؟** قد يمنع جدار الحماية WinRM على الشبكات المصنّفة
> «عامّة». إمّا صنّف شبكتك «خاصّة» (Private)، أو اسمح بالقاعدة:
> `Enable-PSRemoting` عادةً يضيفها للملف الخاص فقط.

---

## 2) في HomeUpdater (جهاز المركز)

1. افتح تبويب **«Windows بعيد»**.
2. اضغط **«إضافة جهاز»** وأدخل:
   - **عنوان IP** للجهاز الهدف (مثال `192.168.1.60`).
   - **المنفذ** `5985` (أو `5986` مع تفعيل HTTPS).
   - **اسم المستخدم**: حساب **مسؤول** على الجهاز الهدف (محلي: `Administrator`،
     أو `اسم-الجهاز\\المستخدم`؛ أو حساب Microsoft/Domain بصيغته الكاملة).
   - **كلمة المرور**.
3. اضغط **«اتصال وحفظ»** — يتحقّق من الاتصال ويكتشف النظام قبل الحفظ (تُخزَّن كلمة
   المرور **مشفَّرة**، ولا تُعرَض أبداً).
4. اضغط **«فحص التحديثات»** ثم **«ترقية الكل»** عند توفّر تحديثات.

---

## 3) استكشاف الأخطاء

| العَرَض | السبب / الحل |
|--------|--------------|
| «تعذّر الوصول إلى WinRM» | WinRM غير مُفعَّل، أو المنفذ 5985 مغلق بجدار الحماية، أو IP خاطئ. أعِد الخطوة 1. |
| «فشل المصادقة» | اسم مستخدم/كلمة مرور خاطئة، أو الحساب ليس مسؤولاً. جرّب `اسم-الجهاز\\Administrator`. |
| «winget غير متوفّر على الجهاز الهدف» | ثبّت **App Installer** من المتجر على الهدف، أو الحساب لا يرى winget في جلسة WinRM. |
| يتصل لكن الترقية ترجع خطأ | بعض الحزم تتطلّب إعادة تشغيل أو تفاعلاً؛ راجع «آخر المخرجات» الظاهرة في البطاقة. |

## 4) اختبار سريع للاتصال من جهاز المركز (اختياري)

```powershell
# يجب أن يطبع اسم الجهاز الهدف لو كان WinRM يعمل والاعتماد صحيح:
$cred = Get-Credential
Invoke-Command -ComputerName 192.168.1.60 -Credential $cred -ScriptBlock { $env:COMPUTERNAME }
```

## ملاحظة أمنية
النقل الافتراضي **NTLM** يُشفّر الحمولة حتى على HTTP (المنفذ 5985)، فبيانات
الاعتماد لا تُرسَل بنصّ صريح. لأقصى حماية استخدم HTTPS (5986) بشهادة على الهدف.
