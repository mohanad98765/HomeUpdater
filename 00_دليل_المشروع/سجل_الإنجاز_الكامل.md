> **الإصدار:** v0.1.0 (وضع `test`)
> **المستودع:** [github.com/mohanad98765/HomeUpdater](https://github.com/mohanad98765/HomeUpdater) (خاص)

## 1. ما هو المنتج

**الرؤية:** برنامج واحد على جهاز واحد في المنزل (Hub)، يكتشف كل الأجهزة المتصلة بالشبكة،
ويُحدّث سوفت‑وير وفِرم‑وير **كل أنواع الأجهزة من مكان واحد**، بإذن المستخدم.

**الواقع الحالي (v0.1.0):** المنتج يعمل فعلياً كتطبيق Windows مُغلَّف (ملف تثبيت واحد)
يكتشف كامل الشبكة، ويُحدّث Windows والبرامج والتعريفات وهواتف Android — لكنه **لم يُغطِّ بعد**
كل أنواع الأجهزة في الرؤية (راوترات/Linux/تلفزيونات/IoT/طابعات). التفصيل في القسم 2.

وحدات تكامل مخصّصة (SSH، OpenWRT API، Home Assistant، CVE feeds) — وهي **العمل القادم**.

## 3. كل ما أُنجز في هذه السلسلة

### 3.1 المراجعة والتصليب (Hardening)
- مراجعة متعدّدة المحاور (backend، frontend، عقد API، كود ميت، أمان، فجوات).
- **حاجز أمان (Release blocker):** الـ backend يعمل بصلاحيات Administrator وواجهته كانت بلا
  مصادقة. أُضيف middleware: قائمة Host بيضاء (يمنع DNS-rebinding) + إلزام ترويسة `X-HomeUpdater`
  على الطلبات المُغيِّرة (يمنع CSRF).

### 3.2 إصلاح عيوب مؤكَّدة
- تعطُّل المسح (500) عند جهازين بلا MAC → العمود `mac` أصبح `nullable`.
- تعطُّل المسح عند **MAC مكرَّر** في نفس الجولة (جهازان بنفس MAC عبر ARP) → دمج بدل الانهيار.
- تحليل `winget` كان يعتمد على عناوين الأعمدة الإنجليزية فيفشل على Windows العربي (يُرجع 0 →
  يُعلِّم الكل «مثبَّتاً») → أُعيد كتابته ليقسِّم على مسافتين ويُثبِّت من اليمين، مُختبَراً عربياً/إنجليزياً.
- سباق في مؤشِّرات التقدّم → قفل. توازن `CoInitialize/CoUninitialize` → موازنة. حقن أوامر ADB
  عبر اسم الحزمة → تحقُّق صارم. أقفال تزامن (409) على المسح/الفحص/التثبيت.
- رسائل الأخطاء المبتلَعة في الواجهة (`detail` مقابل `error`) + تصادم كاش تبويبَي Windows/Drivers.

### 3.3 التنظيف
- حذف 5 وحدات ميتة + ملف `system.py` المؤقّت + اعتماديتَي واجهة، وفصل `requirements.txt`
  إلى ACTIVE/RESERVED. إزالة 8 مفاتيح OUI مكرَّرة كانت تُلبِّس المصنّعين.

### 3.4 الجودة والبنية التحتية
- **65 اختبار pytest** (مُصلِّبة لكل إصلاح: أمان، MAC، winget، ADB، تقدّم، تزامن، migration، اكتشاف).
- **Alembic migrations:** بنية كاملة + migration أساسي، و`init_db` يُشغّل `upgrade head` عند
  الإقلاع (مع تبنّي قاعدة بيانات قديمة عبر `stamp` بلا انهيار).
- **CI (GitHub Actions):** فحص backend (ruff + black + pytest على Windows) + بناء frontend
  (tsc + vite على Ubuntu) — **يمرّ أخضر على خوادم GitHub**.
- تنسيق كامل للـ backend (black + ruff).

### 3.5 التغليف (Phase B — Installer)
- **خادم واحد:** الـ backend يَخدم الواجهة المبنيّة من "/" والـ API من "/api/*".
- **PyInstaller:** `HomeUpdater.exe` واحد (تطبيق tray، بلا نافذة console) يُقلع، يُهاجر،
  يَخدم الواجهة والـ API.
- **خدمة Windows** (`service.py`، pywin32) للتشغيل headless.
- **إشعارات Windows (toast)** عند توفّر تحديثات — عبر أيقونة الـ tray، بلا اعتمادية جديدة.
- **شاشة تحميل (splash)** تظهر فور النقر المزدوج وتبقى حتى يجهز الخادم.
- **مثبِّت Inno Setup:** `HomeUpdater-Setup-0.1.0.exe` (~31MB) بنقرة مزدوجة — اختصارات
  Start Menu + سطح المكتب (افتراضي)، uninstaller، إصدار من `VERSION`.

### 3.6 الهوية البصرية (Phase A)
- علامة افتراضية (منزل أبيض + سهم تحديث فيروزي على خلفية زرقاء متدرّجة) عبر
  `generate_icons.py`، بكل المقاسات + `.ico`، مربوطة بالـ favicon والـ tray وأيقونة الـ exe والمثبِّت.
- بانرات معالج Inno + splash بعربية مُشكَّلة صحيحة (`generate_wizard_banners.py`).
- قابلة لإعادة التوليد من شعار المستخدم لاحقاً.

### 3.7 اكتشاف الشبكة بلا nmap
- بديل **Python خالص** (جسّ TCP لملء جدول ARP ثم قراءته) — لا nmap ولا Npcap ولا صلاحيات
  مدير (يتفادى قيود ترخيص Npcap). nmap يبقى اختيارياً. مُختبَر حيّاً: 23 جهازاً في ~5 ثوانٍ.

### 3.8 إصلاحات وضع windowed
- انهيار عند الإقلاع (`sys.stdout=None` → `isatty`) → حماية المجرى + `log_config=None`.
- تعليق المسح (subprocess يرث stdin غير صالح) → `stdin=DEVNULL` في كل استدعاء.
- **الدرس المنهجي:** اختباراتي السابقة أعادت توجيه المخرجات فأخفت الخللين؛ التحقّق الصحيح عبر
  `Start-Process` (نفس ظرف النقر المزدوج).

### 3.9 ربط GitHub
- مستودع خاص + دفع كل الـ commits + تفعيل CI (مصادقة المستخدم، لا أُدخل بيانات اعتماد).

## 4. المعمارية (كيف يعمل)

```
HomeUpdater.exe (تطبيق tray، windowed)
 └─ uvicorn (خيط خلفي) → FastAPI app
     ├─ يَخدم "/"        → الواجهة المبنيّة (React/Vite) من frontend_dist
     ├─ /api/system/*    → صحّة/إصدار/معلومات/reboot/notify-test
     ├─ /api/devices/*   → اكتشاف الشبكة (Python: TCP+ARP) + قاعدة SQLite
     ├─ /api/updates/*   → Windows Update (COM) + التعريفات + winget
     └─ /api/android/*   → إدارة هواتف Android عبر ADB
 middleware أمان: قائمة Host + إلزام X-HomeUpdater
 قاعدة البيانات: %APPDATA%\HomeUpdater\data\homeupdater.db (Alembic)
 السجلّات:       %APPDATA%\HomeUpdater\logs\
```

- **Backend:** FastAPI + SQLAlchemy async + Alembic (Python 3.12).
- **Frontend:** Vite + React 18 + TypeScript + Tailwind، عربي RTL، ثيمات متعدّدة.

## 7. ما تبقّى (خارطة الطريق)

### أ) لإغلاق v1
- اختبار تثبيت `Setup.exe` على جهاز/VM نظيف.
- توقيع الكود (شهادة) لتجنّب تحذير SmartScreen.
- قلب `build_mode` إلى `release`.

### ب) لتحقيق الرؤية الكاملة (تحديث كل الأجهزة) — Phase 2/3
1. **Linux/SSH** (paramiko): تنفيذ `apt/dnf update` عن بُعد.
2. **راوترات OpenWRT/pfSense:** عبر SSH أو API الراوتر.
3. **Home Assistant integration:** جلب أجهزة المنزل الذكية وتحديثاتها.
4. **CVE feeds (NVD) + GitHub Releases:** عرض الثغرات والتحديثات غير الرسمية.
5. **Android عبر ADB (تحديث فعلي):** بدل فتح Play Store.
6. **إشعارات إيميل/تيليجرام** للتحديثات الأمنية الحرجة.
7. **جدولة تلقائية** للفحص والتحديث (`scan_interval_minutes` موجود كإعداد).

### ج) تحسينات
- إتمام i18n (4 لغات ناقصة)، صفحتا Settings/History، إمكانية الوصول.
- توسيع جدول OUL للمصنّعين (قيد التحسين في مهمّة تدقيق منفصلة).

## 8. ملاحظات وقيود معروفة

- **iOS/Tizen/webOS:** مقفلة من المصنّع — تنبيه فقط (قيد خارجي، ليس نقصاً).
- **Nmap/Npcap:** غير مُضمَّن عمداً (قيود ترخيص) — الاكتشاف يعمل بلغة Python خالصة.
- **قاعدة البيانات:** بعد أي تغيير مخطَّط، احذف `%APPDATA%\HomeUpdater\data\homeupdater.db` مرّة
  واحدة (أو دع Alembic يتبنّاها).
- **الوضع الحالي `test`:** شارة TEST ظاهرة؛ يُقلب لـ `release` عند اعتماد v1.
